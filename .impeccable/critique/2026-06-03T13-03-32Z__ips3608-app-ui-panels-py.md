## What's Working

1. Channel-color discipline (ui_panels.py:173-176). Each MetricCard owns one channel color end-to-end.
2. Delta feedback dual-coded (:143-147). Sign + color. Honors no-info-by-color-alone.
3. Temperature display restraint (:161-206). EMA + 1 Hz refresh kills digit bounce.

## Priority Issues

[P1] Disconnected readout is a silent lie. MetricCard keeps last/zero value on disconnect; 0.00 V off == 0.00 V measured. Violates PRODUCT principle 3. Fix: set_stale() no-data state driven by UiState. -> harden

[P1] Styling literals duplicated instead of tokenized. #4A5A6A ~6x, state hex scattered in inline setStyleSheet. DESIGN.md defines tokens; code ignores them. Drift already present (#16A34A vs documented #22C55E). Fix: central QSS + color constants via objectName. -> extract

[P1] set_connection_state keys off translated display text (:254-269). startswith("connected") breaks on localization. Fix: pass UiState enum, map state->(label,colors). -> harden

[P2] Output toggle has no error-prevention guard. Single button flips 36 V / 8.2 A, no confirm, START/STOP same position. Fix: inline arm/confirm, keep STOP dominant; no modal. -> clarify then harden

[P2] Fan-active uses Voltage Blue on Temperature card (:155). Contradicts DESIGN's own Channel Monopoly Rule. Fix: neutral/state color for fan-on; update DESIGN.md. -> colorize

## Persona Red Flags

Alex (bench power user): no keyboard shortcuts, no focus ring. Must mouse every setpoint. Efficiency gap.
Jordan (first-timer): pre-connect 0.00 V reads as live; no empty state. "Tmax" unexplained vs DESIGN "OTP". Dense one-line status with no emphasis.
Project persona (bench engineer): served well except highest-stakes points — ambiguous no-data, unguarded high-power toggle.
