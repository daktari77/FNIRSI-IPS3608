# Contributing

Thank you for your interest in contributing to IPS3608 Remote Control.
This repository is public and welcomes external contributions, improvements, and fixes.

## How to contribute

1. Fork the repository.
2. Create a new branch for your work:
   - `feature/<short-description>` for new features
   - `fix/<short-description>` for bug fixes
3. Make your changes in a clear, focused commit.
4. Run the existing tests when relevant with:
   ```powershell
   python -m pytest
   ```
5. Open a pull request against the `main` branch.

## Report issues

If you find a bug or want to propose an improvement, please open an issue.
Include a clear description, reproduction steps, and expected behavior.

## Pull request guidelines

- Keep changes small and focused.
- Add tests or documentation updates when appropriate.
- Include a summary of what you changed and why.
- Mention related issues or discussions if applicable.

## Code style

This project uses standard Python practices. Try to keep code readable, simple, and consistent with existing patterns.

## Collaboration

If you want to collaborate directly, feel free to comment on issues or request a review on pull requests.
We are happy to work with contributors on feature ideas, bug fixes, and packaging improvements.
